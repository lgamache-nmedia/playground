CSS Modulaire

Le CSS modulaire, la meilleure façon d'écrire un shitload de css et se retrouver.

Je suis certain qu'on est tous perdus quand on travaille dans le site de quelqu'un d'autre, même si on travaille tous globalement de la même façon. Alors je vais vous proposer quelque chose qui selon moi, sera bien pour tout le monde!

Kossé qu'on fait?

En gros, j'aimerais qu'on essaie un mix entre le OOCSS, BEM et SMACSS. Inquiète-toi pas, je vais tout t'expliquer ça!

OOCSS (Object-Oriented CSS)

On l'utilise pas mal déjà. La clé est simple: au lieu de dire que tous les boutons de ma bannière sont bleus, on va faire un bouton et ajouter une class qui rend le bouton bleu.

Exemple:

Au lieu de :

                
.banner button{
  background: blue;
  padding: 1rem 2rem;
}

Fais :

              
button{
  padding: 1rem 2rem;
}

.blue{
  background: blue;
}

BEM

Je vous en ai déjà parlé mais on l'a jamais essayé (Oli oui mais dans vue.js et c'est inutile car tout est déjà modulaire). Yé temps que ça change!

Dans le fond, BEM c'est seulement un façon de nommer nos classes en suivant le OOCSS. T'as 3 éléments avec BEM: un block, un élément et un modifieur.

Le block

C'est l'élément mère de l'élément. Dans notre exemple, c'est .personCard. Normalement, il est écris en kebab-case mais moi je l'ai mis en camelCase parce qu'on l'utilise déjà partout ailleurs.

L'élément

C'est tout ce qui est dans le block et qui doit rester dans le block. Dans notre exemple, .personCard_text et .personCard_Name sont des éléments. Un élément contient le block suivi du nom de l'élément, séparé par un _. L'élément s'écrit aussi en camelCase.

Le modifieur

Le modifieur sert à modifier notre block. Dans notre exemple, il s'agit de .personCard-blue. Le modifieur contient le block suivi du nom du modifieur, séparé par un -. Le modifieur ne sert qu'à modifier le style d'un block.

SMACSS

SMACSS sert à catégoriser nos classes-objets. Il existe 5 catégories, mais seulement 4 sont utiles.

1. Base

La base est le style général du site web. ce sont les h1, p, ul, wrapper, etc.

2. Layout

Le layout c'est le style qui place les éléments. Un css de layout ne doit pas avoir de style, uniquement de la position.

3. Modules

Les modules sont les styles qui sont appliqués pour un block (notre exemple BEM). Ça devrait être la partie la plus longue du CSS.

4. State

Le state (Ou l'état), sont les classes utiles pour le javascript. par exemple, nous aurions une classe open pour les sous-menus ouverts.

Conventions de nom

SMACSS possède une règle pour certaines catégories:

  • l- précède les règles de layout.
  • m- précède les règles de modules.
  • is- précède les règles de state.

Exemple avec tous ces concepts ensemble

See the Pen Modular CSS exemple by Loïc Gamache (@lo-c-gamache) on CodePen.

Trucs

Lorsque vous éditerez un site qui possède ce système, les seules choses que vous pourrez modifier seront la base et les modules. les states et les layouts seront probablement utilisés par d'autres parties. vous devrez donc créer un autre state ou layout, ou bien ajouter un midifieur au layout.

Conclusion

Avec ce nouveau template, nous pouvons déterminer en un coup d'oeuil ce que la classe veut dire et quels style y seront ajoutés. De plus, lorsque sera venu le temps de modifier le CSS, nous nous retrouverons beaucoup plus rapidement et nous pourront savoir ce que le css impacte réellement. De plus, tout ce concept n'est pas complètement différent de ce que nous faisons déjà.

Local et session storage

Local storage et session storage

Le local storage est vraiment simple et permet de sauvegarder différente données utiles. plus de raison de copier le code de cookie d'un vieux site désuet.

La seule différence entre le local storage et le session storage c'est que le session storage se supprime lorsque tu ferme ton browser. Sinon, le local storage reste jusqu'à ce que tu supprime la cache du browser.

Pour créer un local storage :

                
window.localStorage.setItem('storageName', "storageValue");

Et pour récupérer la valeur: 

                
window.localStorage.getItem('storageName');

Pour faire un session storage, tu fais juste changer local pour session!

Le storage sauvegarde un string alors si tu veux sauvegarder un objet, il est important de le transformer en string avec JSON (Statham).

exemple :

                
var objectName = {
  name: "John",
  lastName: "Doe"
}  

// Set le storage
window.localStorage.setItem('storageName', JSON.stringify(objectName));

// Récupérer le storage
objectName = JSON.parse(window.localStorage.getItem('storageName'));

Font-size fluide

Font-size fluide

Voici le calcul pour avoir des font-size fluides.

              
body {
  font-size: calc([minimum size] + ([maximum size] - [minimum size]) * ((100vw - [minimum viewport width]) / ([maximum viewport width] - [minimum viewport width])));
}

Et maintenant avec les valeurs

              
body {
  font-size: calc(14px + (26 - 14) * ((100vw - 300px) / (1600 - 300)));
}

Dans la source, il y a un mixin de créé. Serait bien de l'ajouter au template de base si ce n'est déjà fait.

Custom drag and drop

Comment bien styliser un drag and drop

Dans certains cas, un site web a besoin d'un drag and drop. Dans tous les cas, c'est chiant à styliser. Mais plus maintenant!

Pour Vsekur, j'avais besoin de styliser un drag and drop assez complexe.

Pour y arriver, j'ai dû utiliser plusieurs sources. Voici un résumé de ces démarches.

Étape 1 : Faire le comportement du drag and drop

Le style de base est déjà appliqué car vous êtes tous capable de le faire (j'espère). J'utilise Sortable.js pour le contrôle du drag and drop. (pour vue.js c'est vue-draggable)

See the Pen Custom drag and drop 1 (Base) by Loïc Gamache (@lo-c-gamache) on CodePen.

Étape 2 : Styliser le placeholder

C'est un bon départ sauf que, je voudrais avoir un meilleur visuel pour l'élément que je déplace afin que ça soit plus clair.

Heureusement pour nous, sortable.js ajoute une classe sortable-ghost lorsque nous déplaçons un objet, alors utilisons cette classe!

See the Pen Custom drag and drop 2 (Placeholder style) by Loïc Gamache (@lo-c-gamache) on CodePen.

Étape 3 : Styliser le ghost

Le ghost est l'élément qui bouge avec la souris. Dans notre cas, on a un design alors il faut le suivre!

En premier, comprenons comment le ghost fonctionne. Le browser prends un genre de screenshot de l'élément que tu déplace dès que tu commence le drag et garde cette image tout le long. J'ai donc essayé la technique de modifier le css pendant quelques millisecondes mais puisque nous avions un autre style pour le placeholder, soit ça ne fonctionnais pas, sois il y avait un glitch visuel. J'ai donc creusé encore un peu.

J'ai finalement trouvé un site obscure qui donnais une solution. En gros, il faut copier l'élément, lui assigner le style en javascript et dire au browser de prendre ce nouvel élément comme image de ghost.

See the Pen Custom drag and drop 3 (Ghost style) by Loïc Gamache (@lo-c-gamache) on CodePen.

Attention, il ne faut pas oublier d'enlever le ghost ajouté précédemment, sinon tu te retrouve avec beaucoup de junk.

Et voilà! Le tour est joué!

Là tu vas me dire : "c'est quoi que ça veut dire tous ces chiffres là!". Le premier c'est la force de transition. à 0, pas de transition, c'est net, frette et sec. Pour un graphique, tu laisse ça à 0. Si tu veux faire autre chose qu'un graphique, amuse-toi! Le deuxième, c'est le pourcentage du rond que tu remplis.

Attention! les pourcentages ne s'additionnes pas automatiquement. Si tu as deux valeurs à 10%, les pointes seront un par dessus l'autre. Pour atteindre le résultat désiré, il faudra avoir la première pointe à 10%, la deuxième à 20%, etc. La deuxième pointe sera donc en dessous de la prmière, mais elle dépassera de 10%.

Courriel à partir d'un site statique

Voici comment faire un envoi de courriel de formulaire à partir d'un site statique en utilisant les Google sheets.

On le sait, il existe des API comme Sendgrid qui peuvent faire la job, mais si on veut pas utiliser ça? J'ai la solution pour toi! Suit ce guide, et tout va bien aller.

Dans le fond, ce qu'on va faire c'est envoyer nos infos de formulaire à un script hébergé sur Google drive, puis ce script va envoyer un courriel et feeder la Google sheet.

Étape 1

Copie ce fichier et renomme-le comme tu veux.

Étape 2

Change les valeurs de la première ligne en correspondance avec les attributs "name" de tes inputs, mais garde le champ "Timestamp". (Tu peux en ajouter/retirer comme tu veux)

Étape 3

Va dans l'éditeur de script, enlève le commentaire à la ligne 8 et change le courriel pour le destinataire du formulaire ( Ex.: info@exemple.com). Pour les tests, tu peux mettre ton adresse courriel, car tu peux changer la valeur quand tu veux.

Étape 4

Il te faudra publier ton script et ce, chaque fois que tu fais une modification. Pour ce faire, enregistre le script puis gère sa version dans "Fichier > Gérer les versions"

Ajoute une description du changement apporté, et fais "Save new version". Tu devra faire ça chaque fois que tu change le script.

Ensuite, il faut publier. Appuie sur "Publier" puis "Déployer en tant qu'application Web...".

Ensuite, sélectionne la version que tu veux déployer. Pour le premier déploiement, sélectionne la version 1, pour le reste, prends une version créée précédemment ou fais juste "nouveau". Soit certain que n'importe qui puisse accéder à l'application, puis copie l'url du script et appuie sur "Mettre à jour".

Google va demander des authorisations, fais juste accepter.

Étape 5

Copie le Javascript du codepen ci-dessous. Tu devra changer quelques sélecteurs, mais rien de trop difficile!

Sur ton formulaire, ajoute aussi l'attribut method="POST" et l'attribut action="TON URL DE SCRIPT" en y collant ton url de script.
